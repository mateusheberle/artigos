## Versões do HTML

1. `HTML`: Começo. *Compartilhar documentos científicos*. Linguagem para focar na estrutura da página.
2. `HTML 2.0`: Web *mais moderna mas simples*.
3. `HTML 3.2` - `HTML 4.0`: Expansão maior da linguagem.
4. `HTML 4.01`: *Tentativa de padronização com uma sintaxe mais rígida*, dificil de ser adotada.
5. `HTML 5`: *Versão recente*

Exemplo: `4.01`
- Versão maior: `4`
- Versão menor: `.01`

**Porque o HTML5 se destacou?**

- *Elementos semânticos*
- *Formulários mais ricos*
- *Multimídia*
- Possibilidade de adicionar *gráficos e animações*
- Ferramentas para *armazenamento de informações no navegador*
- Suporte à *Geolocalização*

**Quem define especificações do HTML?**

*W3C* 
- `World Wide Web Consortium`
- Organização sem fins lucrativos
- *Desenvolver padrões abertos para a web*

**Processo de renderização**

- Garantir que as versões sejam *compatíveis*.
- Os browsers precisam *saber interpretar os novos elementos*.

**Como saber sobre a compatibilidade de um elemento?**

- Consultando documentação
- `MDN Web Docs`

## Estrutura semântica

**O que é uma estrutura semântica?**

- `Semântica`: Ciência que estuda a evolução do *significado das palavras*, signos e símbolos que estão a serviço da comunicação.
- `Estrutura`: HTML

O HTML já possui tags específicas para cada coisa:
- `<header>`
- `<nav>`
- `<main>`
- `<article>`
- `<footer>`

**Exemplo**

```html
<!DOCTYPE html>
<html lang="pt-BR">
    <head>
        <title>Título da página</title>
    </head>
    <body>
        <div id="artigo">
            <h1>Título do artigo</h1>
            <p>Conteúdo do artigo</p>
        </div>
    </body>
</html>
```
```html
<!DOCTYPE html>
<html lang="pt-BR">
    <head>
        <title>Título da página</title>
    </head>
    <body>
        <article>
            <h1>Título do artigo</h1>
            <p>Conteúdo do artigo</p>
        </article>
    </body>
</html>
```

**Por que é importante utilizar tag semântica?**

- **Acessibilidade**: *Leitores de tela* entendem melhor tags semânticas.
- **SEO**: Vão ter um *ranqueamento melhor*.
- **Manutenção**: Mais fácil *dar manutenção*.
- **Futuro**: Já estão preparados para *novas tecnologias*.

### `<header>`

Define o *cabeçalho da página*, geralmente contendo o *título, logo e navegação*. 

### `<nav>`

Representa uma seção de *navegação*, como um *menu*.

### `<main>`

Contém o *conteúdo principal* da página.

### `<article>`

Representa um *conteúdo independente*, como um *post de blog, artigo de notícia* ou comentário.

### `<aside>`

Contém *conteúdo relacionado*, mas não essencial para o fluxo principal do conteúdo.

### `<footer>`

Define o *rodapé da página*, geralmente contendo informações de `copyright`, links para políticas de privacidade, etc.

## Formulários

- *Local onde o usuário fornece informações*. Um exemplo clássico é o *formulário de login e senha*.
- Formulário é talvez a *funcionalidade de integração com o usuário* mais usada na web.
- Sua função é *recolher dados* entrados pelo usuário e *enviá-los ao servidor* para processamento.

### Exemplo

- Quando, em uma página web, colocamos usuário, senha e clicamos em `entrar`. 
- Essa página de formulário está fazendo uma nova solicitação para o servidor, de um outro endereço, sem que precisamos alterar o endereço na barra de `uri`.
- Servidor vai processar a solicitação.
- Alem de um novo endereço, estamos passando um pacote (com o login e senha) também chamado de `payload`.
- Servidor vai processar as informações (trabalho do backend).
- Vai decidir se volta para a mesma página ou não. Uma espécie de negociação.

```html
<form action="processa_formulario.php" method="get">
    <label for="nome">Nome: </label>
    <input type="text" id="nome" name="nome" required>
    
    <label for="email">E-mail: </label>
    <input type="email" id="email" name="email" required>

    <label for="assunto">Assunto: </label>
    <select id="assunto" name="assunto">
        <option value="orçamento">Orçamento</option>
        <option value="outros">Outros</option>
    </select>

    <label for="msg">Mensagem: </label>
    <textarea id="msg" name="massage" cols="50" rows="5"></textarea>
    <button type="submit">Enviar</button>
</form>
```

- `<form> ... </form>`: Usado para criar um *formulário HTML* para entrada do usuário. 
  - Atributos:
    - `action`: *Para onde vai essa informação*.
    - `method`: Por padrão já é `get` (*informação pela URI*), mas pode ser `post` (*informação vai por "baixo dos panos"*).

- `<label> ... </label>`: Define um *rótulo para os elementos* do formulário. Importante para acessibilidade.
  - Atributos:
    - `for`: *Indicar qual elemento está relacionado a ele*, através do `id`. 

- `<input>`: Define um *campo de entrada de dados dentro de um formulário*, podendo ser de diversos tipos, como texto, senha, número, data, e-mail, etc.
  - Atributos:
    - `type`: `text`, `password`, `checkbox`, `number`, `date`, `email` ...
    - `id`
    - `name`
    - `required`: Diz que esse campo é *obrigatório*.
    - `value`: *Valor padrão*.
    - `readonly`: *Somente leitura*.
    - `placeholder`: *Texto pré-preenchido* mas quando o usuário clica ele desaparece.

- `<select> ... </select>`: Define um *menu suspenso* em um formulário, permitindo que o usuário escolha *uma opção de uma lista pré-definida*.
  - Atributos:
    - `id`
    - `name`

- `<option> ... </option>`: Representa *cada uma das opções disponíveis dentro de um menu* `select`, definindo o valor e o texto que será exibido para o usuário.
  - Atributos:
    - `value`: O que vai *ser enviado quando o usuário selecionar essa opção*.
    - `selected`: Informação pré-selecionada.

- `<textarea> ... </textarea>`: Representa uma *área de texto multiline*, permitindo que o usuário insira e edite *grandes quantidades de texto*, como comentários, mensagens ou descrições mais longas.
  - Atributos:
    - `id`
    - `name`
    - `cols`: Quantidade de colunas.
    - `rows`: Quantidade de linhas.

- `<button> ... </button>`: Representa um botão clicável que, ao ser pressionado, executa uma determinada ação, como enviar um formulário.
  - Atributos: 
    - `type`: 
      - `button` (sem nenhuma ação), 
      - `reset` (limpar os dados do formulário), 
      - `submit` (enviar dados ao servidor).

## Validação básica de formulários

Principais atributos para validação:
- `required`: *Torna um campo obrigatório*, exibindo uma mensagem de erro se estiver vazio.
- `minlength`: Especifica o *número mínimo de carcateres* permitidos em um campo.
- `maxlength`: Especifica o *número máximo de caracteres* permitidos em um campo.
- `type`: Define o *tipo de dado esperado em um campo* (text, email, number, etc), ativando validações específicas.

## Glossário

### Compatibilidade de Navegadores
Refere-se à *capacidade dos navegadores da web de interpretar e renderizar corretamente os elementos e funcionalidades do HTML5*, garantindo uma experiência consistente para o usuário.

### Elementos Semânticos
*Tags do HTML5 que fornecem significado ao conteúdo da página*, como "header", "nav", "main", "article", "aside" e "footer", *melhorando a acessibilidade e SEO*.

### Geolocalização
Funcionalidade do HTML5 que permite que uma *página web acesse a localização geográfica do usuário*, geralmente através de APIs de navegador.

### Live Server
*Uma extensão do Visual Studio Code que permite visualizar mudanças em tempo real em uma página web durante o desenvolvimento*, facilitando o processo de teste e iteração.

### MDN (Mozilla Developer Network)
*Uma fonte de documentação abrangente e atualizada para desenvolvedores web*, fornecendo informações sobre HTML, CSS, JavaScript e outras tecnologias da web.

### Multimídia sem Plugins
*Capacidade do HTML5 de incorporar e manipular conteúdo multimídia, como áudio e vídeo*, diretamente no navegador sem a necessidade de plugins externos.

### SEO (Search Engine Optimization)
*Práticas de otimização de páginas web para melhorar sua visibilidade e classificação em motores de busca*, beneficiadas pelo uso de elementos semânticos do HTML5.

### Validações de Formulários
Funcionalidades do HTML5 que *permitem a validação automática de dados inseridos em formulários, melhorando a experiência do usuário e a integridade dos dados* enviados ao servidor.

### W3C (World Wide Web Consortium)
*Organização internacional responsável pelo desenvolvimento e manutenção dos padrões abertos da web*, incluindo o HTML5.

## Reforce seu conhecimento

### O que é HTML5 e por que é importante para o desenvolvimento web moderno?
HTML5 é a quinta versão do HTML, a linguagem padrão para a criação de páginas web. Ele é importante porque *introduziu elementos semânticos*, *suporte a multimídia* sem a necessidade de plugins, e funcionalidades como *armazenamento no navegador* e *geolocalização*, tornando o desenvolvimento web mais eficiente e acessível.

### Qual é o papel do W3C no desenvolvimento do HTML?
O W3C (*World Wide Web Consortium*) é a organização responsável por *desenvolver e manter os padrões abertos da web*, incluindo o HTML. Ele garante que as tecnologias web sejam compatíveis e acessíveis a todos, promovendo a interoperabilidade entre diferentes navegadores e dispositivos.

### Por que a compatibilidade dos navegadores é crucial para o HTML5?
A compatibilidade dos navegadores é crucial para *garantir que os elementos e funcionalidades do HTML5 sejam corretamente interpretados e exibidos*. Isso assegura uma experiência consistente para os usuários, independentemente do navegador que estejam utilizando. Consultar documentações atualizadas, como a do MDN, é recomendado para verificar a compatibilidade.

### Quais são os principais elementos semânticos do HTML5?
Os principais elementos semânticos do HTML5 incluem *"header", "nav", "main", "article", "aside" e "footer"*. Cada um desses elementos tem uma função específica na estrutura de uma página web, melhorando a clareza do código, a acessibilidade e o SEO.

### Como as tags semânticas melhoram a acessibilidade e o SEO?
As tags semânticas melhoram a acessibilidade ao *facilitar a compreensão do conteúdo por tecnologias assistivas*, como leitores de tela. Elas também melhoram o SEO, pois *ajudam os motores de busca a entender a estrutura e o conteúdo da página*, resultando em uma melhor indexação.

### Qual é a importância do `doctype` em um documento HTML5?
O `doctype` é uma declaração que *informa ao navegador que tipo de documento ele está prestes a processar*. Em HTML5, a declaração do `doctype` é simples e garante que o navegador renderize a página de acordo com os padrões HTML5, evitando o modo de compatibilidade.

### Como o HTML5 melhorou a validação de formulários?
O HTML5 introduziu *novos atributos e tipos de `input` que permitem validações automáticas* de formulários, como "required", "email" e "number". Isso melhora a experiência do usuário ao fornecer feedback imediato sobre entradas inválidas, sem a necessidade de scripts adicionais.

### O que é a extensão Live Server e como ela é utilizada?
A extensão Live Server é uma *ferramenta para o Visual Studio Code* que *permite visualizar o progresso de uma página web* em tempo real. Ela cria um servidor local que *atualiza automaticamente a página no navegador* sempre que o código é salvo, facilitando o desenvolvimento e a depuração.

### Como a estrutura semântica do HTML5 beneficia o desenvolvimento web?
A estrutura semântica do HTML5 beneficia o desenvolvimento web ao *tornar o código mais legível e organizado*, *facilitando a manutenção e a colaboração entre desenvolvedores*. Ela também *melhora a acessibilidade e o SEO*, resultando em sites mais eficientes e amigáveis para os usuários.
