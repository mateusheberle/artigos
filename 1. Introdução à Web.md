# Introdução à Web

## Conceitos básicos

**O que é Web?**

- A palavra "Web", vem de *"rede"/"teia"*.

- A web é um *conjunto de arquivos interconectados*.

- Interagimos com essa rede através do *navegador*.

Como a Web funciona: **Cliente**
- Usuário
- Navegador
- Página do Google
- Website escolhido

Como a Web funciona: **Servidor**
- Página do Google
- Conjunto de páginas: Website
- Servidor Web
- Servidor

Host: Servidor Web + Servidor

**Modelo cliente-servidor**:

Dispositivos acessam páginas dentro do servidor.

De um lado:
- Usuário
- Dispositivo
- Navegador
- *Faz uma requisição*

Do lado de quem serve:
- Arquivos
- Servidor web
- Servidor
- *Faz uma resposta*

## Linguagens da Web

3 linguagens:
- HTML
- CSS
- JavaScript

**HTML**
- `HyperText Markup Language` - Linguagem de Marcação de Hipertexto.
- Define a *estrutura* das páginas, dando significado e propósito usando elementos com *títulos*, *parágrafos*, *listas*, *imagens*, *links* e etc.

```html
<p>Isso é um parágrafo</p>
```

**CSS**
- `Cascading Style Sheets` - Folha de Estilos em Cascata.
- Responsável pela aparência *visual* dos elementos, como as *cores*, as *fontes*, os *tamanhos*, *posição* e os estilos em geral. 

```css
p {
    color: red;
}
```
```css
seletor {
    propriedade: valor;
}
```

**JS**
- `Javascript`.
- Adicionar *interatividade* às páginas, como *ações de botões*, *animações*, *jogos*, *formulários dinâmicos* e muito mais.

```js
alert("Olá mundo!");
```

## Estrutura básica de uma página HTML

### Anatomia de um elemento HTML

`<p>Olá Mundo!</p>`: Elemento

- `<p>`: Abertura da tag.
- `Olá mundo!`: Conteúdo.
- `</p>`: Fechamento da tag.

### Aninhando elementos

- `<p>Olá <strong>mundo!</strong></p>`: Elemento pai
- `<strong>mundo!</strong>`: Elemento filho (interno)

### Atributos um elemento HTML

`<p class="destaque">Olá mundo!</p>`

- `class`: Atributo
- `"destaque"`: Valor do atributo

### Elementos vazios

`<img src="minha-foto.png" alt="Minha imagem"/>`: Adicionar uma imagem na página

- `src`: Onde está a imagem
- `alt`: Texto alternativo

## Anatomia de um documento HTML

```html
<!DOCTYPE html>
<html lang="pt-BR">
    <head>
        <title>Título da página</title>
    </head>
    <body>
        ...
    </body>
</html>
```

- `<!DOCTYPE html>`: Declara a versão do HTML5.
- `<html lang="pt-BR">`: Elemento raiz, envolve toda a página.
  - `lang="pt-BR"`: Em qual língua está o conteúdo.
- `<head>`: Informações que não fazem parte do conteúdo.
  - `<title>Título da página</title>`: Título da página.
- `</head>`: Fechamento da tag de informações adicionais.
- `<body>`: Início do conteúdo visível da página.
- `</body>`: Fechamento.

### Cabeçalhos

```html
<!DOCTYPE html>
<html lang="pt-BR">
    <head>
        <title>Título da página</title>
    </head>
    <body>
        <h1>Meu título principal</h1>
        <h2>Meu título de alto nível</h2>
        <h3>Meu subtítulo</h3>
        <h4>Meu segundo subtítulo</h4>
    </body>
</html>
```

- `<H1>` até `<H6>`

### Parágrafo

```html
<!DOCTYPE html>
<html lang="pt-BR">
    <head>
        <title>Título da página</title>
    </head>
    <body>
        <p>Este é um parágrafo simples</p>
    </body>
</html>
```

### Listas não ordenadas

```html

<html lang="pt-BR">
    <head>
        <title>Título da página</title>
    </head>
    <body>
        <ul>
            <li>Item 1</li>
            <li>Item 2</li>
            <li>Item 3</li>
        </ul>
    </body>

```

### Listas ordenadas

```html

<html lang="pt-BR">
    <head>

    </head>
    <body>
        <ol>
            <li>Item 1</li>
            <li>Item 2</li>
```

### Links

```html
<!DOCTYPE html>
<html lang="pt-BR">
    <head>
        <title>Título da página</title>
    </head>
    <body>
        <a href="https://ebaconline.com.br">EBAC Online</a>

```

- `href`: Endereço do link

## Como funciona uma página web

Dividir em 3 partes:
1. Momento que digite o site no navegador e solicita pro servidor.
2. Momento que o servidor interpreta a solicitação e envia os arquivos para o browser.
3. O que o browser faz com esses arquivos.

**O que é uma URI?**

- `Uniform Resource Identifier`.
- Endereço na internet.
- Usado para *identificar de forma única* um recurso na web.
- Pode ser:
  - Uma página web;
  - Uma imagem;
  - Um vídeo;
  - Qualquer outro tipo de arquivo disponível na rede.

`http://127.0.0.1:5500/primeira-pagina.html`
- `http://`: Esquema (schema), é o protocolo de como a comunicação cliente-servidor vai acontecer.
  - `http`
  - `https`: `http` de forma segura
- `127.0.0.1:5500`: Autoridade (domain), endereço IP + Porta.
- `primeira-pagina.html`: Caminho (path)

**Domínio e DNS**

1. Navegador > `google.com` > Internet 
2. Internet > `google.com` > Servidor DNS
3. Servidor DNS > `[2800:3f0:4001:821::2003]` > Internet
4. Internet > `[2800:3f0:4001:821::2003]` > Servidor Web
5. Servidor Web > `index.html` > Internet
6. Internet > `index.html` > Navegador

`Servidor DNS`: Como se fosse uma planilha com nomes e endereços.

**Parsing e renderização**

1. Carrega HTML
2. Analisa HTML
3. Cria estrutura
4. Carrega CSS
5. Analisa CSS
6. Exibe a página

## Glossário

### Cliente-Servidor
Modelo de comunicação na web onde *o cliente (navegador) solicita recursos ou serviços de um servidor, que processa a solicitação e envia uma resposta* de volta ao cliente.

### DNS (Domain Name System)
Sistema que *traduz nomes de domínio legíveis por humanos em endereços IP numéricos*, permitindo que os navegadores localizem e acessem recursos na internet.

### HTML (HyperText Markup Language)
Linguagem de marcação *utilizada para estruturar e organizar o conteúdo de páginas web*, definindo elementos como cabeçalhos, parágrafos e listas.

### HTTP/HTTPS (HyperText Transfer Protocol/Secure)
*Protocolos de comunicação utilizados para a transferência de dados na web*, com HTTPS oferecendo uma camada adicional de segurança através de criptografia.

### JavaScript
Linguagem de programação que *adiciona interatividade e comportamento dinâmico às páginas web*, permitindo a criação de experiências de usuário mais ricas.

### Live Server
Extensão do Visual Studio Code que *permite a visualização em tempo real das alterações feitas no código* HTML, CSS e JavaScript, facilitando o desenvolvimento web.

### URI (Uniform Resource Identifier)
*Sequência de caracteres que identifica de forma única um recurso na web*, como uma página ou um arquivo, permitindo seu acesso através de um navegador.

### Visual Studio Code
*Editor de código-fonte amplamente utilizado para o desenvolvimento web*, oferecendo recursos como depuração, controle de versão e extensões para facilitar a programação.

### Como funciona a interação entre cliente e servidor na web?
A interação entre cliente e servidor na web é baseada no modelo de comunicação cliente-servidor. O cliente (navegador) envia uma solicitação ao servidor, que responde com os documentos solicitados. Essa interação é fundamental para o funcionamento dos sites.

### O que é o Visual Studio Code e por que é importante?
O Visual Studio Code é uma ferramenta crucial para o desenvolvimento de sites. Ele é um editor de código que permite criar e gerenciar arquivos e pastas, facilitando o desenvolvimento web. Familiarizar-se com sua interface é essencial para as etapas subsequentes do curso.

### Quais são as três linguagens principais do desenvolvimento web?
As três linguagens principais do desenvolvimento web são HTML, CSS e JavaScript. HTML estrutura a página, CSS estiliza a aparência visual e JavaScript adiciona interatividade, criando experiências dinâmicas para os usuários.

### O que é a estrutura básica de uma página HTML?
A estrutura básica de uma página HTML inclui elementos como tags de head e body, além de elementos básicos como cabeçalhos, parágrafos e listas. Os alunos aprendem sobre a hierarquia de conteúdo e a definição de atributos.

### O que é a comunicação entre navegador e servidor?
A comunicação entre navegador e servidor envolve o processo de solicitação e resposta de páginas web. Os alunos aprendem sobre URIs, protocolos HTTP e HTTPS, e o papel dos servidores DNS na tradução de nomes de domínio para endereços IP.
